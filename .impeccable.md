# VisionQuest Design Context

## Design Context

### Users
Adults in the SPOKES workforce development program (TANF/SNAP recipients) working toward employment through certifications, goal-setting, and portfolio building. Many have limited digital literacy and varying reading levels. Teachers/instructors use staff tools to guide, monitor, and intervene without chasing information across the program. Students interact primarily through an AI coach named Sage.

### Brand Personality
**Bold, Supportive, Practical** — A direct, action-oriented ally that doesn't patronize. Speaks plainly, celebrates real progress, and keeps the focus on concrete outcomes. Think dependable coach, not corporate software.

### Emotional Goals
- **Confidence**: "I can do this" — clarity about next steps makes the path feel achievable
- **Momentum**: "I'm making progress" — visible evidence that effort is paying off
- **Safety**: "I won't be judged" — a non-threatening space to set goals and ask for help
- **Pride**: "Look what I've done" — celebrating milestones and building a portfolio of evidence

### Aesthetic Direction
- **Reference**: Khan Academy — warm, educational, encouraging, progress-focused without being childish
- **Anti-references**: Corporate LMS platforms (cold, generic), overly gamified apps (patronizing for adults), dense enterprise dashboards (overwhelming)
- **Visual tone**: Warm and grounded with moments of celebration. Mountain-climbing metaphor throughout (progress, milestones, summit). Green = action/growth, Blue = trust/stability, Gold = achievement moments
- **Theme**: Light and dark modes supported. Light is primary.
- **Current stack**: Tailwind CSS 4, Framer Motion, Phosphor Icons, Sora (display) + Manrope (body) fonts

### Accessibility Requirements
- **Standard**: WCAG AA minimum
- **Special focus**: Low literacy users — prioritize visual cues over text, plain language (6th-grade reading level), simple navigation patterns, large touch targets, clear iconography paired with labels
- **Implemented**: Focus-visible states, reduced motion support, skip-to-content link, semantic HTML effort, CSS custom properties for theming

### Design Principles
1. **Clarity over cleverness** — Every screen should answer "what do I do next?" within 3 seconds. Plain language, visual hierarchy, no jargon.
2. **Progress is visible** — Students should always see evidence of forward movement. Use the mountain metaphor, progress bars, streaks, and celebration moments to reinforce momentum.
3. **Warm but not childish** — This serves adults rebuilding their careers. Design should feel encouraging and respectful, never patronizing or gamified in a way that diminishes the user.
4. **Action-first surfaces** — Every view leads to a next step. Minimize passive observation; maximize actionable intelligence for both students and staff.
5. **Inclusive by default** — Design for the lowest common denominator of digital literacy. If a feature requires explanation, simplify the feature, not the explanation.

### Color System Summary
- **Brand green** (#37b550, #2a8a3c): Action, growth, forward momentum
- **Brand blue** (#007baf, #004071): Trust, stability, authority
- **Gold** (#d3b257, #ad8806): Achievement, celebration, special moments
- **Semantic**: Green=success, Blue=info, Gold=warning, Red=error
- **Surfaces**: Layered transparency system with backdrop blur (glassmorphism-lite)
- **Dark mode**: Full CSS variable support, brighter accents for contrast

### Typography Summary
- **Display**: Sora — friendly, modern, distinctive headings (-0.03em tracking)
- **Body**: Manrope — clear, readable, professional
- **Fluid sizing**: clamp() for responsive scaling without breakpoints
- **Low literacy consideration**: Favor larger body text, generous line-height, short line lengths
